## Overview

### What This Plan Accomplishes

This plan delivers a fully functional AI Predictor MVP: a Next.js web application where users paste their LinkedIn URL, wait ~30–60 seconds while the backend scrapes their profile and generates a personalized AI impact report using Claude Sonnet, then view and share that report on LinkedIn. Every component — landing page, processing screen, report generation API, report display page, and LinkedIn share mechanic — is built from scratch in this plan.

### Why This Matters

The MVP is the foundation of the entire product and business. Without a working report, there is no viral loop, no user acquisition, no Authority-AI brand proof point. Speed and quality of the report are the two variables that determine whether this product succeeds. This plan prioritizes getting a working, shareable, high-quality report live as fast as possible.

## Design Decisions

### Key Decisions Made

1. **Synchronous long-poll API pattern**: The `/api/generate-report` endpoint runs the full scrape + generation pipeline synchronously and returns the complete report. The client shows a pre-scripted animated processing screen while awaiting the response. This is simpler than SSE/webhooks and avoids background job infrastructure. Requires `maxDuration = 60` on the route. Vercel Hobby plan allows 60s for serverless functions.

2. **Report stored in Supabase before redirect**: Once generated, the full report JSON is stored in Supabase. The report page fetches from Supabase by username, making reports shareable via URL (anyone can view the report at `/report/[username]` without re-generating). Duplicate requests return the cached report instantly.

3. **Separate `report-status` endpoint**: Allows the processing page to check for an existing report before waiting for a new generation. If a user navigates back or the page refreshes, we don't re-trigger the expensive generation call.

4. **OpenRouter for LLM calls**: Use `https://openrouter.ai/api/v1/chat/completions` with model `anthropic/claude-sonnet-4-6`. This matches the tech stack decision and provides flexibility to swap models. API format is OpenAI-compatible.

5. **Report as structured JSON**: Claude returns the report as a structured JSON object (not markdown prose). This enables rich, section-based UI design rather than rendering a wall of text — better for both readability and sharing.

6. **Research context baked into prompt**: The 4 knowledge base papers' key findings are embedded directly into the system prompt as condensed bullet points. This avoids complex RAG infrastructure for MVP while still grounding the report in real research.

7. **Color scheme — dark, authoritative**: Use a dark navy/slate theme with bright accent (not the authority-ai blue). This signals gravity and expertise appropriate for a career-impact tool, and differentiates from the sibling site.

8. **No authentication**: Reports are public by LinkedIn username. No login required. The trade-off is potential abuse, but for MVP virality > friction. Rate limiting is a post-MVP concern.

9. **Apify actor**: Use `curious_coder~linkedin-profile-scraper` (a well-maintained community actor). The actor is called synchronously via Apify's REST API: start run → poll for completion → fetch results. This may need to be updated to whatever actor is most reliable at implementation time.

### Alternatives Considered

- **SSE streaming progress**: More accurate real-time feedback but significantly more complex. The fake pre-scripted animation is simpler and still feels engaging. Can upgrade post-MVP.
- **n8n webhooks (like authority-ai)**: Would require deploying and managing a separate n8n workflow. Unnecessary complexity for MVP — direct API calls are simpler.
- **Markdown report output**: Simpler to generate but harder to style richly. JSON structure enables better UI design.
- **RAG over PDFs**: Full vector search over research PDFs. Better long-term, but overkill for 4 papers. Embedding key findings in the prompt is sufficient for MVP.

### Open Questions (if any)

1. **Apify actor ID**: Confirm which LinkedIn Profile Scraper actor is currently most reliable on Apify. At implementation time, verify the actor ID and its expected output schema. Consider: `curious_coder~linkedin-profile-scraper` or `apify/linkedin-profile-scraper`.
2. **AI Exposure Score scale**: The plan uses 1–10. Should it be displayed as a number, a letter grade (A–F), or a label ("High/Medium/Low Risk")? Recommendation: show the number AND a label for maximum shareability.
3. **Rate limiting**: For MVP launch, what's acceptable? Recommendation: no rate limiting for MVP — monitor cost and add limits if abuse occurs.

  const SUPABASE_URL = process.env.NEXT_PUBLIC_SUPABASE_URL!
  const SUPABASE_SERVICE_ROLE_KEY = process.env.SUPABASE_SERVICE_ROLE_KEY!

  async function supabaseRequest<T>(
    path: string,
    options: RequestInit = {}
  ): Promise<T> {
    const url = `${SUPABASE_URL}/rest/v1/${path}`
    const response = await fetch(url, {
      ...options,
      headers: {
        apikey: SUPABASE_SERVICE_ROLE_KEY,
        Authorization: `Bearer ${SUPABASE_SERVICE_ROLE_KEY}`,
        'Content-Type': 'application/json',
        Prefer: 'return=representation',
        ...options.headers,
      },
      cache: 'no-store',
    })

    if (!response.ok) {
      const error = await response.text()
      throw new Error(`Supabase error ${response.status}: ${error}`)
    }

    const text = await response.text()
    return text ? JSON.parse(text) : null
  }

  export async function getReport(username: string) {
    const rows = await supabaseRequest<any[]>(
      `reports?username=eq.${encodeURIComponent(username)}&limit=1`
    )
    return rows?.[0] ?? null
  }

  export async function insertReport(data: {
    username: string
    linkedin_url: string
    status: string
  }) {
    const rows = await supabaseRequest<any[]>('reports', {
      method: 'POST',
      body: JSON.stringify(data),
    })
    return rows?.[0] ?? null
  }

  export async function updateReport(
    username: string,
    data: {
      status: string
      report?: object
      profile_data?: object
      error_message?: string
    }
  ) {
    await supabaseRequest(
      `reports?username=eq.${encodeURIComponent(username)}`,
      {
        method: 'PATCH',
        body: JSON.stringify(data),
      }
    )
  }

### Step 7: Create Apify LinkedIn Scraper Helper

Build the helper that calls Apify to scrape a LinkedIn profile.

**Actions:**

- Create `src/lib/apify.ts`:
  ```typescript
  import type { LinkedInProfile } from '@/types/report'

  const APIFY_API_KEY = process.env.APIFY_API_KEY!
  // Verify this actor ID at implementation time — check Apify marketplace
  // for "LinkedIn Profile Scraper". Recommended: curious_coder~linkedin-profile-scraper
  const ACTOR_ID = 'curious_coder~linkedin-profile-scraper'
  const BASE_URL = 'https://api.apify.com/v2'

  interface ApifyRunResponse {
    data: {
      id: string
      status: string
      defaultDatasetId: string
    }
  }

  interface ApifyStatusResponse {
    data: {
      status: 'READY' | 'RUNNING' | 'SUCCEEDED' | 'FAILED' | 'TIMED-OUT' | 'ABORTED'
    }
  }

  async function startRun(linkedinUrl: string): Promise<string> {
    const response = await fetch(
      `${BASE_URL}/acts/${ACTOR_ID}/runs?token=${APIFY_API_KEY}`,
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          startUrls: [{ url: linkedinUrl }],
          proxy: { useApifyProxy: true },
        }),
        cache: 'no-store',
      }
    )

    if (!response.ok) {
      throw new Error(`Apify run start failed: ${response.status}`)
    }

    const data: ApifyRunResponse = await response.json()
    return data.data.id
  }

  async function waitForRun(
    runId: string,
    maxWaitMs = 30000,
    pollIntervalMs = 2000
  ): Promise<string> {
    const deadline = Date.now() + maxWaitMs
    while (Date.now() < deadline) {
      await new Promise((resolve) => setTimeout(resolve, pollIntervalMs))

      const response = await fetch(
        `${BASE_URL}/actor-runs/${runId}?token=${APIFY_API_KEY}`,
        { cache: 'no-store' }
      )
      const data: ApifyStatusResponse = await response.json()
      const status = data.data.status

      if (status === 'SUCCEEDED') return runId
      if (['FAILED', 'TIMED-OUT', 'ABORTED'].includes(status)) {
        throw new Error(`Apify run ${status.toLowerCase()}`)
      }
    }
    throw new Error('Apify run timed out waiting for completion')
  }

  async function getRunResults(runId: string): Promise<any[]> {
    const response = await fetch(
      `${BASE_URL}/actor-runs/${runId}/dataset/items?token=${APIFY_API_KEY}&clean=true`,
      { cache: 'no-store' }
    )

    if (!response.ok) {
      throw new Error(`Failed to fetch Apify results: ${response.status}`)
    }

    return response.json()
  }

  function normalizeProfile(raw: any): LinkedInProfile {
    // Normalize Apify output to our LinkedInProfile type
    // Field names may vary by actor version — adjust as needed
    const experience = (raw.positions?.positionHistory ?? raw.experience ?? []).map(
      (pos: any) => ({
        title: pos.title ?? pos.jobTitle ?? '',
        company: pos.companyName ?? pos.company ?? '',
        duration: pos.timePeriod?.duration ?? pos.duration ?? '',
        description: pos.description ?? '',
      })
    )

    const skills = (raw.skills ?? []).map(
      (s: any) => (typeof s === 'string' ? s : s.name ?? '')
    )

    return {
      fullName: raw.fullName ?? raw.name ?? '',
      headline: raw.headline ?? '',
      summary: raw.summary ?? raw.about ?? '',
      location: raw.addressWithCountry ?? raw.location ?? '',
      industry: raw.industryName ?? raw.industry ?? '',
      currentRole: experience[0]?.title ?? raw.headline ?? '',
      currentCompany: experience[0]?.company ?? '',
      yearsExperience: raw.yearsOfExperience ?? experience.length * 2,
      skills: skills.slice(0, 20),
      experience: experience.slice(0, 5),
      education: (raw.educations ?? raw.education ?? []).map((edu: any) => ({
        school: edu.schoolName ?? edu.school ?? '',
        degree: edu.degreeName ?? edu.degree ?? '',
        field: edu.fieldOfStudy ?? '',
      })),
    }
  }

  export async function scrapeLinkedInProfile(
    linkedinUrl: string
  ): Promise<LinkedInProfile> {
    const runId = await startRun(linkedinUrl)
    await waitForRun(runId)
    const results = await getRunResults(runId)

    if (!results || results.length === 0) {
      throw new Error('No profile data returned from Apify')
    }

    return normalizeProfile(results[0])
  }
  ```

### Step 9: Build the Generate Report API Route

The main API endpoint that orchestrates the full report generation pipeline.

**Actions:**

- Create `src/app/api/generate-report/route.ts`:
  ```typescript
  import { NextRequest, NextResponse } from 'next/server'
  import { scrapeLinkedInProfile } from '@/lib/apify'
  import { generateReport } from '@/lib/openrouter'
  import { getReport, insertReport, updateReport } from '@/lib/supabase'

  export const maxDuration = 60

  const LINKEDIN_REGEX =
    /^https?:\/\/(www\.)?linkedin\.com\/in\/([a-zA-Z0-9\-_%]+)\/?([?#].*)?$/i

  function extractUsername(url: string): string | null {
    const match = url.trim().match(LINKEDIN_REGEX)
    if (!match?.[2]) return null
    return decodeURIComponent(match[2]).toLowerCase().replace(/\/$/, '')
  }

  export async function POST(request: NextRequest) {
    try {
      const body = await request.json()
      const linkedinUrl = typeof body?.linkedin_url === 'string'
        ? body.linkedin_url.trim()
        : ''

      const username = extractUsername(linkedinUrl)
      if (!username) {
        return NextResponse.json(
          { error: 'Please enter a valid LinkedIn profile URL.' },
          { status: 400 }
        )
      }

      // Check for existing complete report
      const existing = await getReport(username)
      if (existing?.status === 'complete' && existing?.report) {
        return NextResponse.json({
          ok: true,
          username,
          status: 'complete',
          cached: true,
        })
      }

      // Check if currently processing (avoid duplicate runs)
      if (existing?.status === 'processing') {
        return NextResponse.json({
          ok: true,
          username,
          status: 'processing',
          cached: false,
        })
      }

      // Insert or update row as processing
      if (!existing) {
        await insertReport({ username, linkedin_url: linkedinUrl, status: 'processing' })
      } else {
        await updateReport(username, { status: 'processing' })
      }

      // Step 1: Scrape LinkedIn profile
      const profileData = await scrapeLinkedInProfile(linkedinUrl)

      // Step 2: Generate report
      const report = await generateReport(profileData)

      // Step 3: Store in Supabase
      await updateReport(username, {
        status: 'complete',
        profile_data: profileData,
        report,
      })

      return NextResponse.json({
        ok: true,
        username,
        status: 'complete',
        cached: false,
      })
    } catch (error) {
      console.error('generate-report error:', error)
      const message = error instanceof Error ? error.message : 'Unknown error'

      // Try to mark the report as errored if we have a username
      try {
        const body = await request.clone().json()
        const url = body?.linkedin_url ?? ''
        const username = extractUsername(url)
        if (username) {
          await updateReport(username, { status: 'error', error_message: message })
        }
      } catch {
        // Ignore secondary errors
      }

      return NextResponse.json(
        { error: 'Failed to generate report. Please try again.' },
        { status: 500 }
      )
    }
  }
  ```

  import { useEffect, useState } from 'react'
  import { useRouter } from 'next/navigation'
  import { CheckCircle2, Circle, Loader2 } from 'lucide-react'

  const STEPS = [
    {
      id: 'scraping',
      label: 'Fetching your LinkedIn profile',
      detail: 'Reading your career history, skills, and experience...',
      duration: 8000,
    },
    {
      id: 'analyzing',

      detail: 'Cross-referencing McKinsey, Stanford, and Anthropic data...',
      duration: 12000,
    },
    {
      id: 'generating',
      label: 'Generating your personalized report',
      detail: 'Building your timeline, risks, and action plan...',
      duration: 15000,
    },
    {
      id: 'finalizing',
      label: 'Finalizing your report',
      detail: 'Almost ready...',
      duration: 5000,
    },
  ]

  type StepStatus = 'pending' | 'active' | 'complete'

  interface ProcessingScreenProps {
    username: string
    linkedinUrl: string
  }

  export function ProcessingScreen({ username, linkedinUrl }: ProcessingScreenProps) {
    const router = useRouter()
    const [stepStatuses, setStepStatuses] = useState<Record<string, StepStatus>>(
      Object.fromEntries(STEPS.map((s, i) => [s.id, i === 0 ? 'active' : 'pending']))
    )
    const [apiDone, setApiDone] = useState(false)

    // Animate steps based on durations
    useEffect(() => {
      let elapsed = 0
      const timers: ReturnType<typeof setTimeout>[] = []

      STEPS.forEach((step, index) => {
        // Mark step as active at its start time
        const startTimer = setTimeout(() => {
          setStepStatuses((prev) => ({ ...prev, [step.id]: 'active' }))
        }, elapsed)
        timers.push(startTimer)

        elapsed += step.duration

        // Mark step as complete at its end time
        const endTimer = setTimeout(() => {
          setStepStatuses((prev) => ({ ...prev, [step.id]: 'complete' }))
        }, elapsed)
        timers.push(endTimer)
      })

      return () => timers.forEach(clearTimeout)
    }, [])

    // Fire the actual API call
    useEffect(() => {
      async function generate() {
        try {
          const response = await fetch('/api/generate-report', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ linkedin_url: linkedinUrl }),
          })
          const data = await response.json()

          if (data.ok) {
            setApiDone(true)
          } else {
            router.push(`/?error=${encodeURIComponent(data.error ?? 'Generation failed')}`)
          }
        } catch {
          router.push('/?error=Something+went+wrong')
        }
      }

      generate()
    }, [linkedinUrl, router])

    // Navigate to report once API is done AND animation is at least on last step
    useEffect(() => {
      if (!apiDone) return

      // Check if all steps are complete; if so, redirect immediately
      // Otherwise wait for last step to be active before redirecting
      const allComplete = STEPS.every((s) => stepStatuses[s.id] === 'complete')
      const lastStepActive =
        stepStatuses[STEPS[STEPS.length - 1].id] === 'active' ||
        stepStatuses[STEPS[STEPS.length - 1].id] === 'complete'

      if (allComplete || lastStepActive) {
        // Give a brief moment on last step before redirecting
        const timer = setTimeout(() => {
          router.push(`/report/${username}`)
        }, 1500)
        return () => clearTimeout(timer)
      }
    }, [apiDone, stepStatuses, username, router])

    return (
      <div className="min-h-screen flex flex-col items-center justify-center px-6 py-20">
        <div className="w-full max-w-md">
          <div className="mb-10 text-center">
            <h1 className="text-2xl font-bold text-text">Analyzing your career</h1>
            <p className="mt-2 text-text-muted text-sm">
              This takes about 60 seconds. Please don&apos;t close this tab.
            </p>
          </div>

            {STEPS.map((step) => {
              const status = stepStatuses[step.id]
              return (
                <div key={step.id} className="flex items-start gap-4">
                  <div className="mt-0.5 shrink-0">
                    {status === 'complete' ? (
                      <CheckCircle2 className="h-5 w-5 text-success" />
                    ) : status === 'active' ? (
                      <Loader2 className="h-5 w-5 text-accent animate-spin" />
                    ) : (
                      <Circle className="h-5 w-5 text-border" />
